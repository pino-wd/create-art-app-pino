# v1.1.0 修复补遗 — Art Design Pro 完整搬迁

## Context

v1.1.0 首轮修复已通过 lint/type-check/smoke-test，但代码审查发现 5 个问题：
1. 完全移除 sass 过于激进，Art Design Pro 的组件大量使用 SCSS
2. 登录页未使用 Art Design Pro 的完整组件（LoginLeftView、AuthTopBar、ArtDragVerify）
3. 异常页 SVG 插画是简陋占位符，应使用 Art Design Pro 原始 SVG
4. Layout 布局页未使用 Art Design Pro 的组件（ArtSidebarMenu、ArtHeaderBar、ArtPageContent）
5. Dashboard 页面未使用 Art Design Pro 的 Console 仪表盘

**用户决策：允许引入 Art Design Pro 完整生态**，包括 vue-i18n、menuStore、settingStore、hooks、config、locales 等。尽量从 Art Design Pro 直接搬迁，不做简化。

## 关键适配点

1. **路径别名**：Art Design Pro 使用 `@imgs/`、`@styles/` 等别名，需在 vite.config.ts 添加
2. **SCSS additionalData**：注入 `@use "@styles/core/el-light.scss"` 和 `@use "@styles/core/mixin.scss"`
3. **`__APP_VERSION__`**：StorageConfig 依赖此全局变量，需在 vite.config.ts define 中声明
4. **fastEnter.ts**：路由名称需替换为脚手架中存在的路由
5. **component.ts**：全局组件配置简化，去除设置面板、锁屏等不需要的组件
6. **Dashboard 与 echarts**：Dashboard 模块使用 echarts，需确保 feature-echarts 被默认选中或 echarts 加入 base 依赖
